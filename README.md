Unity Performance Optimization Project
이 프로젝트는 유니티 환경에서 대규모 객체와 UI 리스트를 효율적으로 관리하기 위한 최적화 기법을 적용한 사례입니다.

주요 구현 기능
1. Dialog System 및 Prefab 연결
Prefab 기반 관리: UI 다이얼로그를 독립적인 프리팹으로 제작하여 독립성을 확보하고 재사용성을 높였습니다.

Scene 연결: 메인 게임 루프와 다이얼로그 시스템 사이의 데이터 전달 및 활성화를 스크립트로 제어하여 씬 전환 시의 효율성을 개선했습니다.

2. On-Demand 방식의 Object Pooling (Initial Pool: 0)
무한히 생성되는 적 캐릭터의 인스턴스화 부하를 해결하기 위해 동적 확장형 오브젝트 풀링을 적용했습니다.

런타임 동적 생성: 적이 스폰될 때 풀이 비어있다면 즉시 새 객체를 생성하여 사용하고, 이후 풀에 등록합니다.

재활용 및 최적화: 적 사망 시 Destroy 대신 SetActive(false) 처리 후 풀에 반환합니다. 게임이 진행될수록 생성 빈도는 줄어들고 재활용 빈도가 높아져 시스템이 안정화됩니다.

3. RecycleView 방식의 스테이지 리스트 최적화
스테이지 선택 화면에서 대량의 데이터를 효율적으로 보여주기 위해 화면 양옆의 예비 객체를 활용한 재배치 시스템을 구현했습니다.

예비 화면 생성: 현재 화면을 기준으로 이전 화면(왼쪽)과 다음 화면(오른쪽)에 대응하는 UI 요소들을 미리 생성해둡니다.

실시간 재배치 알고리즘: 사용자가 화면을 넘기면, 시야에서 벗어난 이전 화면의 요소들을 즉시 반대편(진행 방향의 앞쪽)으로 이동시킵니다.

데이터 바인딩: 이동된 객체는 파괴되지 않고 위치 값과 내부 데이터만 업데이트되어 끊임없이 이어지는 리스트를 구현합니다.

성능 결과: 전체 스테이지 개수와 상관없이 일정한 수의 UI 객체만 유지하므로 메모리 사용량이 고정되며 스크롤이 매끄럽습니다.
